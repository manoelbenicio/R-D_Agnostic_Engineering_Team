Isolamento de Credencial por Conta — Estado Alvo

Cada conta vive isolada. O daemon resolve a conta do agente, restaura a credencial AS-IS no diretório próprio e injeta a env var nativa do vendor. Logar/renovar uma conta não afeta as outras. Fase 2 troca de conta automaticamente ao esgotar a janela de ~5h.

Solução

Clique em um componente

Ciano = tarefa; roxo = resolução da conta; azul = preparação/injeção; verde/lima/amarelo = credenciais isoladas por conta; vermelho = laço de rotação (Fase 2); laranja = leitura/escrita no Postgres.

Chave: cópia por conta (não symlink global) → o refresh de uma conta nunca contamina outra.
Cirúrgico: só o caminho de auth muda; orquestração/canvas/dispatch permanecem intocados.
Concorrência: uma credencial válida pode servir N agentes durante seu ciclo (~5h).
