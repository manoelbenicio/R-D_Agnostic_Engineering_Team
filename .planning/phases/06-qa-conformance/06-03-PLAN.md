# P6-03 — Implement L2 Sidecar in Rust to Unblock QA Conformance

**Objective:** P6-01 and P6-02 are BLOCKED because nothing listens on `127.0.0.1:43117` with the `rpp.l2.v1` control surface. This plan implements a minimal Rust L2 sidecar (REQ-29) so the C1-C6 smoke tests can run against a real process.

**Output:** Rust sidecar binary, Go launch configuration, C1-C4 smoke evidence, 06-03 summary.

**Context:** `docs/contracts/l2-runtime-contract.md`, `multica-auth-work/server/internal/l2runtime/client.go`, `multica-auth-work/server/internal/daemon/l2_runtime.go`.

## Tasks

1. **Minimal Rust sidecar (prodex-runtime-broker binary)**  
   Add an HTTP server binary to the `prodex-runtime-broker` crate that binds `127.0.0.1:43117` and exposes:
   - `GET /healthz`
   - `GET /readyz` with `contract_version=rpp.l2.v1`, `status=ready`, `checks` containing `shared_state_backend=pass`
   - `POST /v1/policy/apply`
   - `POST /v1/accounts/register` (fail-closed on invalid `profile_home`)
   - `POST /v1/session/start` (returns `router_owner=rust_l2`, `event_stream_url`)
   - `POST /v1/session/stop`
   - `GET /v1/events/stream` (NDJSON events with `secrets_present=false`)

2. **Wire Go daemon launch**  
   Update `multica-auth-work/server/.env` so `MULTICA_PRODEX_PATH` points to the sidecar binary and `MULTICA_L2_SIDECAR_ARGS` starts the server. Confirm `multica-server` can launch it and `readyz` responds.

3. **Rerun C1-C4 smokes and capture evidence**  
   Execute `readyz-smoke.sh`, `policy-apply-smoke.sh`, `session-start-stop-smoke.sh`, `event-stream-smoke.sh`, `profile-fail-closed-smoke.sh` with `SMOKE_ALLOW_EXECUTE=1`, recording scrubbed output in `.deploy-control/evidence/`.

4. **Container Go gate**  
   Run `go build ./... && go vet ./internal/... && go test ./internal/daemon ./internal/l2runtime -count=1` inside `golang:1.26-alpine` with IPv6 disabled.
