This document provides a comprehensive mapping of **Phase 3 (Skills System)**, **Phase 4 (Settings — All 10 Tabs)**, and **Phase 7 (Search)** for the Multica platform. It includes structural descriptions, component mappings, form field schemas, validation rules, user permissions logic, and screenshot references.

## 1. Skills System

### 1.1 Skills Overview
The Skills system is Multica's core differentiator, allowing users to bundle specific instructions, prompt templates, and custom rules into reusable abilities that can be assigned to one or more AI agents. 

- **Access Path:** Configure → Skills (`/configure/skills` or `/[workspaceSlug]/skills`).
- **Permissions:** 
  - Anyone in the workspace can view skills.
  - Only the skill creator or a workspace Administrator/Owner can edit or delete a skill.
  - Changes made to a skill take effect dynamically on the next agent run.

### 1.2 Skill Properties & Schema
A skill consists of the following data structures and files:
- **Name:** Unique string identifier within the workspace. Used by agents to reference the skill.
- **Description:** One-sentence summary of when and how an agent should use this skill.
- **Main Instructions (`SKILL.md`):** A mandatory markdown file that defines the system instructions, prompts, and behaviors for the skill.
- **Supporting Files:** Additional custom files (e.g., templates, context data) nested within the skill package.
- **Metadata:** 
  - **ID:** Autogenerated unique database identifier.
  - **Origin:** Details on where the skill came from (Manual, Local Runtime, ClawHub, Skills.sh, or GitHub).
  - **Created By:** Reference to the user who added/created the skill.
  - **Created At / Updated At:** ISO timestamps tracking creation and modifications.

### 1.3 Skill Creation Flow
Clicking **"+ New skill"** opens a creation dialog with three distinct methods:
1. **Create Manually:**
   - **Name Input:** Placeholder `e.g. review-helper`. Must be unique. Capitalization and symbols normalized. Validation fails if name conflict exists.
   - **Description Textarea:** Placeholder `One sentence on when to assign this skill to an agent.`
   - Submitting creates the skill and redirects the user to the Skill Detail editor.
2. **Import from URL:**
   - **URL Input:** Input field with placeholder `https://clawhub.ai/owner/skill`.
   - **Source Detection:** The system automatically analyzes the domain of the URL to detect if it's from **ClawHub** (`clawhub.ai`), **Skills.sh** (`skills.sh`), or **GitHub** (`github.com`).
   - Clicking **"Import"** fetches the skill package server-side.
3. **Copy from Runtime:**
   - Promotes a skill already installed on a connected local runtime into the workspace.
   - **Runtime Selection:** A dropdown displays available local runtimes owned by the user. The selected runtime must be online.
   - **Skills List:** Renders checkboxes next to all discoverable on-disk skills for the active runtime.
   - **Inline Edit:** Selecting exactly one skill expands the row to allow editing the *Workspace skill name* and *Description* before importing.
   - **Batch Import:** Users can select multiple skills and import them in bulk (executed concurrently up to 10 threads).
   - **Conflict Handling:** If a name conflict occurs, a resolution card appears with options:
     - *Overwrite:* Overwrites the workspace skill (only allowed if the user is the creator or an admin).
     - *Rename:* Prompts for a new name (defaults to adding ` copy` suffix).
     - *Skip:* Bypasses the skill.

### 1.4 Skill Detail & Editing Flow
The skill detail page (`/[workspaceSlug]/skills/[id]`) features a robust three-column layout:
1. **Left Sidebar - File Tree:**
   - Lists files included in the skill. `SKILL.md` is mandatory and cannot be deleted.
   - **"+ Add file"** button: Opens an inline input with placeholder `templates/review.md` to add a new file. Validates that the path is relative, does not contain `..`, is not `SKILL.md`, and does not already exist.
   - **"Delete file"** button: Appears at the bottom of the tree when a custom file is selected.
2. **Center Pane - Editor:**
   - **Name Input:** Inline editable text header. Locked with a lock icon if the user does not have edit permissions.
   - **Description Textarea:** Editable description box.
   - **Metadata Subline:** Displays origin source, relative updated time (via `useTimeAgo`), and the creator's avatar + name.
   - **Conflict Invalidation Banner:** If another user edits the skill while this view is active, a warning banner appears: *"Someone else updated this skill. Your edits are preserved. Discard to pull their changes, or Save to overwrite."*
   - **Code Editor:** Displays the content of the selected file. Supports markdown preview or raw text mode.
   - **Save Bar:** Stays anchored at the bottom of the editor when changes are detected:
     - Displays: *"Unsaved changes — will overwrite the live skill on save"*
     - Buttons: *Discard* (reverts to server state) and *Save changes* (commits edits).
3. **Right Sidebar - Metadata & Assignments:**
   - **Metadata:** Displays ID, Origin info, Created, and Updated dates.
   - **Permissions Info:** Text block describing user rights.
   - **Used By section:** Lists all agents assigned to this skill. If unassigned, displays: *"Not assigned to any agent yet. Open an agent's Skills tab to assign."*

### 1.5 Skill-Agent Assignment
- Skills can be added to agents directly from the Skills page toolbar via the **"Actions"** row dropdown.
- Shows list of available agents grouped into **"My agents"** and **"Other agents"**.
- Displays fractional count badges (e.g. `1/2 added`) if a skill is partially assigned.
- Selecting multiple skills triggers a **Batch Toolbar** at the bottom with options to Add to Agent or Delete in bulk.

## 2. Settings — General
- **Access Path:** Settings → General (`/settings?tab=general` or `/[workspaceSlug]/settings?tab=workspace`).
- **Fields:**
  - **Logo Upload:** A square button showing the first letter of the workspace name (or logo if uploaded). Hovering reveals a camera icon overlay. Opens file picker. Accepts `.png`, `.jpeg`, `.webp`. Only accessible to Admin/Owner roles.
  - **Name Input:** Workspace name.
  - **Description Textarea:** Placeholder `What does this workspace focus on?`.
  - **Context Textarea:** Background information and context passed to AI agents working on issues. Placeholder `Background information and context for AI agents working in this workspace`.
  - **Slug Input:** Displays the current URL path slug (Read-only).
  - **Issue Prefix Input:** Uppercase alphanumeric string (max 10 characters). Defines prefix for issue identifiers (e.g., `MUL` for `MUL-123`). 
- **Validation & Confirms:**
  - Changing the issue prefix triggers a warnings dialog: *"All issues will be renumbered from [OLD]-N to [NEW]-N. External references — pull request titles, branch names, links in docs and chat — that use [OLD]-N will stop resolving."*
- **Danger Zone:**
  - **Leave Workspace Button:** Removes user access. Disabled for the sole owner: *"You're the only owner. Promote another member to owner first, or delete the workspace."*
  - **Delete Workspace Button:** Only visible to Owners. Opens a typed confirmation modal (GitHub-style) requiring the user to type the exact workspace name to enable the **"Delete workspace"** button.

## 3. Settings — Members
- **Access Path:** Settings → Members (`/settings?tab=members`).
- **Invite Member Form:**
  - Standard text input for invitee's email.
  - Dropdown to choose role: **Member** (create and work on issues) or **Admin** (manage members and settings).
  - **"Invite"** button: Sends an invitation and adds it to the "Pending invitations" list.
- **Members List:**
  - Lists current workspace members showing avatar, name, email, role badge, and join date.
  - **Role Management:** Dropdown trigger on member rows to demote or promote roles (**Owner**, **Admin**, **Member**).
  - **Owner Constraint:** Demoting the last Owner is blocked: *"Promote another member to owner first — a workspace must keep at least one owner."*
  - **Remove Member:** Triggers a confirmation dialog to revoke workspace access.
- **Pending Invitations List:**
  - Shows pending invite email, pending status tag, role badge, and a **"Revoke"** (X) button.

## 4. Settings — Repositories
- **Access Path:** Settings → Repositories (`/settings?tab=repositories`).
- **Description:** Lists Git repositories associated with this workspace. AI agents clone and inspect code from these repositories.
- **List and Management:**
  - Displays Repository URL and Description.
  - Row controls: **Edit** (Pencil) and **Delete** (Trash) buttons. Editing expands the row into editable inputs.
  - **"+ Add repository"** button: appends a blank row at the end of the list.
  - **"Save"** button: saves all repository changes in one operation. Disabled if no changes exist.

## 5. Settings — GitHub Integration
- **Access Path:** Settings → GitHub (`/settings?tab=github`).
- **Toggles and Connection:**
  - **Enable GitHub Features Master Switch:** Toggles the integration. If OFF, all GitHub features are hidden across the workspace.
  - **GitHub App Connection Card:**
    - Shows connection status: *Connected to [AccountName]* or *GitHub is not connected*.
    - **"Connect GitHub"** button: opens GitHub App authorization page in a new tab.
    - **"Disconnect"** button: opens disconnect confirmation dialog.
- **Features Configuration:**
  - **Pull Request Sidebar Switch:** Toggle to show linked pull requests inside the issue details sidebar.
  - **Co-authored-by Trailer Switch:** Append `Co-authored-by: multica-agent <github@multica.ai>` trailers to commits authored by agents.
  - **Auto-link Issues and PRs Switch:** Matches PR titles/bodies/branches against issue identifiers (`MUL-123`) to link them automatically.
- **Repositories Shortcut:**
  - A card containing a button link pointing directly back to the Repositories tab.

## 6. Settings — Integrations (Lark)
- **Access Path:** Settings → Integrations (`/settings?tab=integrations`).
- **Lark Bot Binding Flow:**
  - Shows Lark integration description. Supports Lark (international) and Feishu (mainland China).
  - If Lark environment variables (`MULTICA_LARK_SECRET_KEY`) are missing on the server, it displays a disabled card stating the integration is not enabled.
- **Connected Bots List:**
  - Lists connected bots bound to agents.
  - Displays agent avatar, agent name, region tag (Feishu/Lark), installation timestamp, and a **"Disconnect"** button.
  - Disconnecting tears down the WebSocket connection.
- **Binding Dialog (LarkInstallDialog):**
  - Triggered by clicking "Bind to Lark" from an Agent's details page.
  - Generates a QR code and session ID. Polls the server until authorized by the user.
  - Shows: Scan instructions, fallback browser link, scan status (e.g. QR code expired, network retry, access denied).

## 7. Settings — Profile
- **Access Path:** Settings → Profile (`/settings?tab=profile`).
- **Personal Profile Details:**
  - **Avatar Upload:** Clickable circular profile avatar displaying initials if empty. Hover reveals camera icon.
  - **Name Input:** Personal display name.
  - **"About you" Textarea:** Detailed profile description shared directly with agents working on behalf of this user. Capped at 2000 characters. Shows real-time character count.
  - **"Update Profile" Button:** Saves the profile settings.

## 8. Settings — Preferences
- **Access Path:** Settings → Preferences (`/settings?tab=preferences`).
- **Theme Selection:**
  - Radio card buttons for **Light**, **Dark**, and **System** themes.
  - System theme preview diagonal mock window splits Light/Dark.
- **Language Selector:**
  - Options: **English**, **中文 (Chinese)**, **한국어 (Korean)**, **日本語 (Japanese)**.
  - Changing language writes a local cookie, patches the user settings on the backend, and reloads the page.
- **Timezone Selector:**
  - Timezone selector dropdown containing full IANA timezone list.
  - Default value is `__browser__` (browser timezone).
  - Used for dashboards, metrics, and dates displayed in the UI.

## 9. Settings — Notifications
- **Access Path:** Settings → Notifications (`/settings?tab=notifications`).
- **Inbox Notifications:**
  - Toggles for categories:

## 12. Search

### 12.1 Search Interface
- **Trigger:** Pressed globally via `Cmd+K` or `Ctrl+K`. Or by clicking the Search button in the layout sidebar.
- **Command Palette:** Centered dialog overlay built using Radix Dialog + `cmdk` primitive.
- **Input:** Standard search text input with placeholder `Type a command or search...` and an `ESC` keyboard badge.
- **Keyboard Navigation:** Arrow keys select list items. Enter triggers selection. Escape closes the palette.

### 12.2 Search Result Types
Search results are grouped dynamically based on the query:
1. **Pages:** Fast navigation links pointing to Inbox, My Issues, Issues, Projects, Agents, Runtimes, Skills, and Settings.
2. **Commands:** Actions like: *New Issue*, *New Project*, *Copy Issue Link* (only shown if currently visiting an issue page), *Copy Identifier*, and *Theme toggles*.
3. **Members:** Workspace members matching the query name, email, or role. Supports Pinyin matching for Chinese names.
4. **Projects:** Projects matching name or description, showing active status badge.
5. **Issues:** Issues matching title, description, identifier, or comments. Shows ID, status icon, title, assignee avatar, and text snippets containing matching queries.
6. **Recent:** If the query is empty, displays the recently visited issues list.

### 12.3 Autocomplete and Highlights
- Queries are debounced by 300ms to reduce database loading.
- API requests use `AbortController` to cancel in-flight requests if typing continues.
- All matches in labels and text snippets are highlighted using the `HighlightText` component.
